# Documentation

## Docstrings
- Use **Google-style docstrings**.
- Generate for public APIs, services, and models.
- Omit for private methods, simple getters/setters, or test functions.

## README & Architecture
- Update `README.md` endpoints, features, and structures when changing code.
- Update architecture docs in `documentation/` for significant changes (e.g., `CLOUD_RUN_FRONTEND_ARCHITECTURE.md`).
